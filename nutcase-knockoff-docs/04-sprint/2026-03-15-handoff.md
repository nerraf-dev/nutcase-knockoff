## Target UX Flow
1. Home -> Setup (`game_init`)
- Pick game basics only (type, length/target, maybe mode later).
- Continue.

2. Setup -> Lobby
- Show room code and connection instructions.
- Show live joined players.
- Allow host actions: start, leave, kick.
- Start enabled only when minimum conditions met (>=2 connected and ready).

3. Lobby -> Game Board
- Only at start click from lobby host.

## Recommended Minimal Implementation Order

### 1) Add lobby transition in main
- In `main.gd`, change `_on_game_init_complete(settings)` to:
  - cache settings (member var),
  - load lobby scene,
  - do **not** call `GameManager.start_game(settings)` yet.

### 2) Create lobby contract (signals)
In `lobby.gd`, add signals:
- `lobby_start_requested(settings: Dictionary)`
- `lobby_back_to_home`
- optional: `lobby_back_to_setup`

Main should connect these and decide scene transitions.

### 3) Start/stop server at lobby lifecycle
- On lobby enter: set `NetworkManager.is_local = false` and `start_server()`.
- On leaving lobby/home: `stop_server()`.
- Render `NetworkManager.room_code`.

### 4) Handle join/ready roster
In lobby, connect NetworkManager signals:
- `player_join_received(device_id, player_name, avatar_index)`
- `player_ready_received(device_id)`
- `client_disconnected(device_id)`

Maintain a lobby-ready map and refresh list UI on every event.

### 5) Gate start button
Enable Start only when all are true:
- player count >= 2
- required players ready (recommend all connected ready)

### 6) Start game from lobby
On start press:
- emit `lobby_start_requested(cached_settings)` to main.
- In main handler:
  - `cleanup_current_scene()`
  - call `GameManager.start_game(settings)`
  - `load_game_board()`

## Open Design Decisions (Pick early)

- Kick behavior: remove from `PlayerManager` immediately or mark disconnected.
- Whether setup keeps `player_count` for MP; likely infer from lobby roster at start.
