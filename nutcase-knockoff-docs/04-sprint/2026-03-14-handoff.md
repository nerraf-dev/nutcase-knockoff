Objective: support host + phone join before game start.

Tasks:

- Stop server on leaving lobby/game over or returning to menu.

### 3) Implement join/ready handling end to end
Objective: map network join events to PlayerManager and gate start by readiness.

Tasks:
- Handle NetworkManager.player_join_received:
  - Validate unique name/device_id
  - Create or update Player with device_id/avatar_index/is_connected
- Handle NetworkManager.player_ready_received:
  - Track ready status in lobby state
- Broadcast roster updates to all controllers.
- When game starts, broadcast game_started.

Definition of done:
- Multiple controllers can join and mark ready; host start respects readiness rules.

### 4) Make start_game multiplayer-safe
Objective: avoid creating placeholder local players when lobby has real players.

Tasks:
- Add mode-aware branch in GameManager.start_game():
  - Local mode: current behavior (create Player 1..N)
  - Multiplayer mode: use already joined players from lobby; do not recreate
- Keep clear_all_players logic for local restarts, but do not wipe valid lobby roster during MP transition.

Definition of done:
- Starting MP game preserves joined players and their metadata.

### 5) Broadcast gameplay updates to controllers
Objective: keep remote UI synchronized.

Tasks:
- On round start: broadcast_new_round(round_num, slider_count)
- On turn change: broadcast_turn_changed(player_id) and direct your_turn to active player
- On reveal: broadcast_slider_revealed(index, word, revealer_id)
- On score updates: broadcast_scores(players)
- On fuzzy vote: broadcast_vote_request and broadcast_vote_result
- On round/game completion: broadcast_round_end and broadcast_game_over

Definition of done:
- Controller UI can run purely from server push events for a full round.
