1. Voting clarity and pacing
- Add a short 2-step beat in UX copy: “Vote incoming” then “Cast your vote now”.
- Keep timing predictable: 1.0-1.5s pre-vote reveal, then full vote window.
- Put the rule on screen every time during vote: “Majority accepts. Tie = reject.”
- After vote closes, always show one explicit outcome line: “Accepted” or “Rejected, no-voters split X”.

2. Voting tension without extra systems
- Use a simple countdown language rhythm: “3… 2… 1… Vote”.
- Make controller instruction ultra-specific:
  - “Tap YES if close enough”
  - “Tap NO if not acceptable”
- Avoid dense text in this phase; one action per screen.

3. Scoring balance (your 7-question win feeling)
- For a 200 target, 7 perfect questions can feel slightly short in 3-player sessions. *This was 7 questions in total, to be clear, answered by different players, it was an 'evenly matched' game.*
- Fast tuning options to test in playtests:
  - Raise target to 250 for 3+ players, or
  - Reduce early bonus by 1-2 points, or
  - Keep points but add more interruption events (freeze/vote moments) to create drama. *Freeze and voting only happen on answering, so it feels like something new would be needed. Have ideas, but all new so more work.*
- Best quick test: run 3 matches each with target 200 vs 250 and compare average match length + “felt tension” score from players.

4. Keep interest through tile identity (without big redesign)
- Treat tiles as visual stakes:
  - “Unrevealed = pressure”
  - “Revealed = information gained”
- Add post-round copy that references board state, not just points:
  - “You won early, most tiles still hidden.”
  - “Late solve, board was almost fully revealed.”
- This keeps the tile mechanic emotionally relevant even before adding “smash/overlay” mechanics later. *When noted i was thinking more of some sort of visual representation of 'winning' rather than just the number on the screen. Maybe the closer to winning, the more your own tile (player badge) cracks - you want to crack and smash it to 'reveal the winning you' or something cheesy. That or closer to winning you are, more cracks others get.*

5. Game-end emoji issue
- Good catch for builds.
- For v1, use plain text symbols or words only:
  - “Winner!”
  - “Champion”
  - “Game Over”
- Avoid emoji dependency in any critical message path.

  - Total rounds to win
  - Number of vote events
